Migración mini.deps → lazy.nvim + UI Modernization

3 enfoques para modernizar tu editor. Clickeá el que más te convenza.

A

🎨 Snacks Takeover + Premium UI

Máximo eye candy. Snacks.nvim para toda la UI core + plugins premium para statusline y tabline.

✓ Ganas

  • Dashboard animado con ASCII art
  • Picker moderno con previews estilizados
  • Notificaciones animadas y elegantes
  • Indent guides animados con scope highlighting
  • Smooth scroll + scrollbar
  • Zen mode, dim inactive code
  • bufferline.nvim — tabline con iconos, colores, separadores curvos
  • lualine.nvim — statusline con secciones, iconos, themes Catppuccin
  • noice.nvim — cmdline floating, mensajes estilizados, LSP progress

✗ Trade-offs

  • Más plugins que mantener (~5 nuevos)
  • Mayor complejidad de config
  • Posible impacto en startup time (mitigado con lazy-loading)

Reemplaza: mini.notify → snacks.notifier · mini.pick → snacks.picker · mini.starter → snacks.dashboard · mini.files → snacks.explorer · mini.indentscope → snacks.indent · mini.map → snacks.scroll · mini.statusline → lualine · mini.tabline → bufferline · mini.clue → which-key · mini.bufremove → snacks.bufdelete · mini.cursorword → snacks.words

Mantiene mini.nvim: ai · surround · operators · pairs · align · move · splitjoin · jump · jump2d · bracketed · comment · diff · git · hipatterns · sessions · visits · snippets · trailspace · icons · misc · basics

B

⚡ Snacks Core + Mini Text

Balance óptimo. Snacks para toda la UI visible, mini.nvim solo para text manipulation.

✓ Ganas

  • Todo lo visual de snacks (dashboard, picker, notify, indent, scroll)
  • snacks.statuscolumn para columna lateral
  • Zen mode + dim + image support
  • Menos plugins externos que A (sin lualine/bufferline/noice)
  • Statusline y tabline quedan en mini (funcionales, no premium)

✗ Trade-offs

  • Statusline/tabline siguen con look mini (menos visual)
  • Sin cmdline floating (noice)
  • El look no es 100% cohesivo — mix de estilos snacks + mini

Reemplaza: mini.notify → snacks.notifier · mini.pick → snacks.picker · mini.starter → snacks.dashboard · mini.files → snacks.explorer · mini.indentscope → snacks.indent · mini.map → snacks.scroll · mini.clue → which-key · mini.bufremove → snacks.bufdelete · mini.cursorword → snacks.words

Mantiene mini.nvim (UI): statusline · tabline · (todo lo de text manipulation de A)

C

🔧 Selective Upgrade

Mínima disrupción. Solo reemplaza los 4-5 componentes de mayor impacto visual.

✓ Ganas

  • Dashboard con snacks (mayor impacto visual)
  • Notificaciones animadas
  • Smooth scroll
  • Migración más fácil y rápida
  • Menor riesgo de romper workflows

✗ Trade-offs

  • Picker queda en mini.pick (funcional pero menos visual)
  • File explorer queda en mini.files
  • No es una modernización completa
  • Mix de estilos más notorio

Reemplaza: mini.starter → snacks.dashboard · mini.notify → snacks.notifier · mini.map → snacks.scroll · mini.indentscope → snacks.indent

Mantiene todo lo demás de mini.nvim (picker, files, statusline, tabline, clue, etc.)

💡 Recomendación

Enfoque A es el que mejor se alinea con lo que pediste: UI moderna, cohesiva, eye candy máximo. La combinación snacks + bufferline + lualine + noice es el stack más popular del ecosistema Neovim para configs visualmente pulidas. Todos tienen soporte nativo para Catppuccin.

Los ~21 módulos de mini.nvim que quedan (text manipulation, git, sessions, etc.) no tienen mejor alternativa — son excelentes y no afectan la UI. Es el mejor de ambos mundos.
