Cambio clave: De 2 archivos monolíticos (30_mini.lua + 40_plugins.lua) a archivos modulares por plugin. Cada plugin spec es autocontenido — config, keymaps, y lazy-loading en un solo lugar.

🔄 Fases de Migración

Cada fase deja el editor funcional. Se puede pausar entre fases.

FASE 0 Preparación
  • Crear branch feat/lazy-migration
  • Agregar .superpowers/ a .gitignore
  • Crear lua/config/ — extraer options, keymaps, autocmds de plugin/
  • Crear lua/plugins/ vacío
FASE 1 Package Manager (mini.deps → lazy.nvim) — fundación, todo depende de esto
  • Reescribir init.lua con bootstrap de lazy.nvim
  • Convertir todos los add() / now() / later() a lazy specs
  • Migrar mini.nvim a specs individuales por módulo
  • Migrar plugins existentes (treesitter, LSP, conform, blink, etc.)
  • Resultado: mismo editor, nuevo package manager — :Lazy funciona
FASE 2 UI Shell (statusline + tabline + cmdline)
  • Instalar y configurar lualine.nvim con theme Catppuccin
  • Instalar y configurar bufferline.nvim con Catppuccin
  • Instalar y configurar noice.nvim (cmdline floating, LSP progress)
  • Remover mini.statusline y mini.tabline
  • Resultado: statusline, tabline y cmdline modernos
FASE 3 snacks.nvim (core UI + features nuevos) — mayor impacto visual
  • Instalar snacks.nvim y configurar todos los módulos:
  • Reemplazos: dashboard, notifier, picker, explorer, indent, scroll, bufdelete, words
  • Nuevos: animate, zen, dim, image, input, rename, lazygit, terminal, statuscolumn, toggle
  • Remover módulos mini reemplazados
  • Adaptar pick-utils.lua para snacks.picker
  • Reemplazar float-term.lua con snacks.terminal
  • Reemplazar toggle.lua con snacks.toggle
  • Resultado: toda la UI usando snacks — look cohesivo
FASE 4 which-key.nvim + Keymaps
  • Instalar which-key.nvim con groups e iconos
  • Migrar descripciones de mini.clue
  • Migrar keymaps de plugins a sus respectivos lazy specs
  • Remover mini.clue
  • Resultado: key hints modernos y keymaps colocados
FASE 5 Cleanup & Polish
  • Eliminar archivos obsoletos (plugin/30_mini.lua, plugin/40_plugins.lua, mini-deps-snap)
  • Eliminar utils reemplazados (float-term.lua, toggle.lua)
  • Verificar que Catppuccin integra bien con todos los plugins nuevos
  • Ajustar highlights y colores para cohesión
  • Profile startup time con :Lazy profile
  • Resultado: config limpia, moderna, cohesiva

⚡ Estrategia de seguridad

  • Todo en un branch feat/lazy-migration — main queda intacto
  • Cada fase termina con un commit funcional — se puede pausar o revertir
  • Los keymaps se mantienen idénticos — solo cambia el backend
  • mini.nvim se queda instalado como dependencia de lazy.nvim (no se borra)
